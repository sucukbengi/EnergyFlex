EnergyFlex

Forecast building load and smart‑schedule devices to minimize peaks and bills. Built with Next.js + FastAPI + Celery + OR‑Tools + TimescaleDB.

View on GitHub See the Demo Architecture

Why I built this

Small buildings face peak demand charges and time‑of‑use tariffs but lack enterprise EMS tools. EnergyFlex predicts demand and optimizes device schedules to reduce cost and carbon without sacrificing comfort.

Highlights

  • End‑to‑end web app with async optimization
  • CP‑SAT model with capacity constraints
  • Time‑series DB & seeded readings
  • Dockerized + CI workflow

Demo

Run locally with Docker:

docker compose up --build

UI: http://localhost:3001 · API: http://localhost:8001/health · DB: localhost:5434

Architecture

Next.js (UI)  →  FastAPI (REST)  →  Redis (broker/result)
     ↑                  ↓
  Browser polls    Celery worker  →  OR‑Tools CP‑SAT
     ↑                  ↓
   Charts        TimescaleDB (meter_readings, schedules, tasks)

Flow: 1) forecast → 2) optimize → 3) worker writes back results → 4) UI lists saved schedules.

Optimization details

Variables: integer start times, interval constraints, per‑hour load. Constraints: device windows, duration, capacity (≤ peak). Objective: minimize peak (can extend to cost under TOU). CP‑SAT fits this discrete scheduling problem well.
